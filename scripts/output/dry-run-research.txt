DRY-RUN ANALYSIS — ISSUES FOUND + FIXES APPLIED
============================================================

ISSUE 1: Sunbiz name matching — false positives
  Problem: search_sunbiz_local() was checking if words appeared
  *anywhere* in the full cordata line (including address fields).
  Example: "10 Palms North Townhomes" matched "INDEPENDENT RESOURCE
  FACILITATION, LLC" whose address happened to contain "PALMS".
  Result: wrong entity — wrong doc#, wrong status, wrong address.

  FIX (applied to script):
  - Match only against entity name field (cols 13–93 of record).
  - All search words must appear in the 80-char name field only.
  - Added more stop words to reduce single-word ambiguity.
  - Added minimum 2-word requirement for name matching.

ISSUE 2: Zillow fee range slider noise
  Problem: extract_from_text() was capturing "$50/mo" through
  "$2000/mo" from Zillow's fee range filter UI, not real HOA fees.
  Result: 13 junk fee observations per community.

  FIX (applied to script):
  - Fee regex now requires explicit HOA context within 100 chars:
    "HOA fee", "monthly HOA", "HOA dues", "HOA assessment".
  - Bare "$XXX/month" only counted if preceded by HOA-related text.
  - Minimum fee raised from $50 to $75 to filter small slider values.

ISSUE 3: New DB columns not yet live
  Problem: communities table doesn't yet have gated, age_restricted,
  entity_status, state_entity_number, registered_agent, street_address,
  zip_code, incorporation_date, monthly_fee_max, monthly_fee_median,
  website_url, etc. (migration 20260502_pending_data.sql not run yet).
  The select query was updated to only query existing columns.

  FIX (applied to script):
  - DB select uses only confirmed-existing columns.
  - Auto-approve writes still attempt to update new columns —
    these will fail gracefully until migration is run in production.

RECOMMENDATION: Run supabase/migrations/20260502_pending_data.sql
in Supabase SQL Editor before running in live mode.
